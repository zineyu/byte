# Data Model: Automatic Compaction Entry

**Feature**: specs/002-auto-compaction-entry  
**Date**: 2026-07-16

## Entities

### Compaction Entry

A `CompactionEntry` is a durable, persisted summary node that replaces a contiguous block of older active-path messages when constructing LLM context.

| Field | Type | Description | Validation |
|-------|------|-------------|------------|
| `id` | `String` | Stable identifier, unique within the Session | Non-empty, UUID or opaque id |
| `role` | `MessageRole::Summary` | Fixed role for LLM context reconstruction | Must be `Summary` |
| `summary` | `String` | Natural-language summary generated by the Model Provider | Non-empty, length bounded by budget policy |
| `compacted_range` | `CompactionRange` | Identifies the oldest contiguous block of active-path messages being summarized | Must reference existing messages |
| `created_at` | `String` | ISO 8601 timestamp of creation | Valid timestamp |
| `run_id` | `String` | The Run during which the Compaction Entry was created | References an existing Run |

`CompactionRange` contains:

| Field | Type | Description |
|-------|------|-------------|
| `first_message_id` | `String` | First message in the compacted block |
| `last_message_id` | `String` | Last message in the compacted block |

### Session Entry

The persisted `SessionEntry` enum in `byte-session` is extended with a new variant:

```rust
pub enum SessionEntry {
    Session { ... },
    Message(Message),
    SkillActivated(ActivatedSkill),
    CompactionEntry(CompactionEntry),   // NEW
}
```

The `Message` type already supports `MessageRole::Summary` in the UI view, but the persisted authority is the `CompactionEntry` variant.

### SessionView

The UI-facing `SessionView` already contains `messages: Vec<Message>`. Compaction Entries are rendered as `Message` nodes with `role: MessageRole::Summary`. The UI distinguishes them from regular messages by role and renders them as expandable timeline items.

## Relationships

```text
Session (1) ──contains──▶ CompactionEntry (0..*)
Session (1) ──contains──▶ Message (0..*)
CompactionEntry (1) ──summarizes──▶ Message (1..*)
Run (1) ──produces──▶ CompactionEntry (0..1)
```

## State Transitions

### Run with Compaction

```text
[Idle]
  │ Developer sends message
  ▼
[RunStarted] ──RuntimeEvent(RunStarted)──▶ UI
  │
  ▼ active path ≥ 90% budget
[CompactionNeeded]
  │
  ├── emit RuntimeEvent(CompactionStarted)
  │
  ▼
[Summarization] ──call Model Provider via byte-models──▶ Summary text
  │
  ▼ summary received and persisted
[CompactionCompleted] ──RuntimeEvent(CompactionCompleted)──▶ UI
  │
  ▼
[ModelTurn] ──active path now includes CompactionEntry──▶ Model Provider
  │
  ▼
[RunFinished] ──RuntimeEvent(RunFinished)──▶ UI
```

### Failure Transitions

| From | Condition | To | Event/Action |
|------|-----------|-----|--------------|
| `CompactionNeeded` | No prior active-path messages to summarize | `RunFailed` | `RuntimeEvent(Error { run_id, message: "context budget exceeded with no prior history" })` |
| `Summarization` | Single message exceeds budget | `RunFailed` | `RuntimeEvent(Error { run_id, message: "message exceeds context budget" })` |
| `Summarization` | Summary itself exceeds budget after compaction | `RunFailed` | `RuntimeEvent(Error { run_id, message: "compaction did not reduce context below budget" })` |
| `Summarization` | Run cancelled while compaction in progress | `RunCancelled` | Persist partial state if any; allow later Run |

## Active Path Reconstruction

1. Load all `SessionEntry` records from JSONL.
2. Walk the active path from the newest leaf backward to the root, collecting `Message` and `CompactionEntry` nodes.
3. When a `CompactionEntry` is encountered, include it in the active path and skip all `Message` nodes whose `id` falls within its `compacted_range`.
4. The resulting active path is a linear sequence of recent `Message` nodes and `CompactionEntry` nodes, ordered by creation time.
5. When constructing the LLM context, translate each `CompactionEntry` into an `LlmMessage` with `role: MessageRole::Summary` and body containing the summary text.

## Determinism Rules

- Compaction is triggered by the same token threshold (90%).
- The oldest contiguous block is selected deterministically by message order.
- The summarization prompt is deterministic given the same message block.
- The Model Provider may be non-deterministic; the feature treats the returned summary as authoritative, but the *decision* to compact and the *range* selected are deterministic.
